# Security policy

## Supported versions

The latest prerelease and the current `main` branch receive security fixes during the public beta. Older commits and prereleases are unsupported.

## Reporting a vulnerability

Use [GitHub private vulnerability reporting](https://github.com/micobruh/nl-jobflow/security/advisories/new). Do not open a public issue for a vulnerability before it is resolved.

Include the affected command, impact, reproduction steps using fictional data, and the smallest safe diagnostic output. Never send a real CV, `.env`, configuration, SQLite database, Telegram token, application document, or other candidate data.

This is a local single-user tool. Reports should identify a concrete boundary an external vacancy, website, dependency, or local process can cross; ordinary access by the same OS user is not considered a privilege boundary.

## User responsibilities

Keep profiles private, review `/doctor` permission warnings, rotate exposed credentials, and inspect generated drafts before using them. The project provides advisory screening, not legal or immigration advice.
